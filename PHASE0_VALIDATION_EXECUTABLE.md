## **✅ Étape 7 : Vérifier les données détectées par Grype/Trivy**

```powershell
# Lire les résultats Grype
$GRYPE_FILE = "C:\Users\user\Desktop\medianet\scan-results\$SCAN_ID\grype.json"

# Pour chaque CVE, vérifier:
# - cveId
# - artifact.name
# - artifact.version
# - artifact.purl
# - artifact.type

Get-Content $GRYPE_FILE | ConvertFrom-Json | ForEach-Object {
    $_.matches | Select-Object @{
        Name="CVE"; Expression={$_.vulnerability.id}
    }, @{
        Name="Package"; Expression={$_.artifact.name}
    }, @{
        Name="Version"; Expression={$_.artifact.version}
    }, @{
        Name="PURL"; Expression={$_.artifact.purl}
    } | Format-Table -AutoSize
}
```

## **✅ Étape 10 : Calculer les métriques SBOM**

```powershell
# Terminal 6 - PowerShell

$SCAN_ID = "VOTRE_UUID"
$DB_QUERY = @"
SELECT
    COUNT(*) as total_cves,
    SUM(CASE WHEN component_name IS NOT NULL THEN 1 ELSE 0 END) as matched_components,
    SUM(CASE WHEN direct_or_transitive IN ('DIRECT', 'TRANSITIVE') THEN 1 ELSE 0 END) as relations_resolved,
    SUM(CASE WHEN direct_or_transitive = 'UNKNOWN' THEN 1 ELSE 0 END) as unresolved
FROM cve_entries
WHERE scan_id = '$SCAN_ID';
"@

# Exécuter avec psql:
psql -U medianet_user -d medianet_db -c $DB_QUERY
```

**Format de résultat attendu:**

```
total_cves | matched_components | relations_resolved | unresolved
-----------|-------------------|-------------------|----------
80         | 45                | 35                | 45
```

**Calculer:**

```
Taux composants matchés = 45 / 80 = 56.25%
Taux relations résolues = 35 / 80 = 43.75%
Taux non-résolus = 45 / 80 = 56.25%
```

## **🚨 Troubleshooting si problèmes**

### **Problème: Backend ne démarre pas**

```powershell
# Vérifier les logs
# Chercher: "error", "exception", "failed to initialize"
# Solution: Check application.properties, PostgreSQL connection
```

### **Problème: SBOM n'existe pas**

```powershell
# Vérifier que scanner a terminé
# Check: scan-results/UUID/result.json
# Si absent: Scanner en train de tourner, attendre
# Si présent mais SBOM absent: Ajouter étape SBOM dans pipeline Docker
```

### **Problème: Success rate = 0%**

```
Possibles:
1. SBOM absent → Ajouter génération SBOM
2. Versions UNKNOWN → Extraire depuis Maven
3. PURL null → Vérifier extraction Grype/Trivy
```
